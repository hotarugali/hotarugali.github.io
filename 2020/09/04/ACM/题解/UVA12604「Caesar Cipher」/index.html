UVA12604「Caesar Cipher」

发表于2020-09-04|更新于2021-04-11|ACM题解
|字数总计:810|阅读时长:4分钟|阅读量:

1. 题目

题目链接：UVA12604「Caesar Cipher」 。

Description

In cryptography, a Caesar cipher, also known as Caesar’s cipher, the shift cipher, Caesar’s code or Caesar shift, is one of the simplest and most widely known
encryption techniques. It is a type of substitution cipher in which each letter in the plaintext is replaced by a letter some fixed number of positions down the
alphabet (wrapping around in the end). For example, given an alphabet of capital letters in usual order, with a shift of 333, AAA would be replaced by DDD, BBB would become EEE, and so on, with ZZZ being replaced by CCC. The method is named after Julius C Caesar, who used it in his private correspondence.
  We are given an alphabet AAA, a string SSS which is encrypted using a shift cipher and a plaintext word WWW. Find the possible values of shifts (in the range [0,∣A∣−1][0, |A|-1][0,∣A∣−1]) used in encryption if we know that the unencrypted text contains exactly one occurrence of the word WWW.

Input

Input starts with an integer NNN on a line, the number of test cases. Each cases contains three strings on separate lines, alphabet AAA, plaintext word WWW and encrypted text SSS. Alphabet AAA will contain only letters and digits ([‘A’-‘Z’][‘a’-‘z’][‘0’-‘9’]) and its symbol order is not necessarily lexicographical (see the third sample case). AAA will not contain duplicate symbols. The constraints are as given: 3≤∣A∣≤62;1≤∣W∣≤50,000;3≤∣S∣≤500,0003 \leq |A| \leq 62; 1 \leq |W| \leq 50,000; 3 \leq |S| \leq 500,0003≤∣A∣≤62;1≤∣W∣≤50,000;3≤∣S∣≤500,000 .

Output

For each test case print one line of output.

  • If there are no shifts that would satisfy the condition of WWW being a part of the unencrypted SSS, print ‘no solution’.
  • If there is exactly one shift that could have been used, print ‘unique: #’ where # is the shift value.
  • It there are more than one possible shifts print 'ambiguous: ’ followed by the sorted list of shift values.

For clarification, see the sample output.

2. 题解

分析

显然是一道 KMP 题，由于 ∣A∣|A|∣A∣ 较小，故考虑直接枚举 WWW 加密后所有可能的密文，即枚举 shift=0..∣A∣−1 shift = 0..|A|-1 shift=0..∣A∣−1，然后对每个 WWW 的密文应用 KMP 算法，查找其在主串 SSS 中匹配的次数，只有次数为 1 时表示此 shiftshiftshift 有效，然后统计所有有效的 shiftshiftshift 即可。
